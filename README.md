# PHP Browser

**Десктопный браузер для локальных PHP-лендингов** — сканирует папки на диске, поднимает встроенный PHP-сервер, показывает превью в webview, редактирует код, заменяет видео и даёт встроенный терминал прямо в папке лендинга.

Сделано для команд, которые работают с сотнями affiliate-лендингов на macOS: быстрый поиск, группировка по корневым директориям, кириллические пути, история правок и HLS-видео без ручного ffmpeg.

## Возможности

### Навигация и поиск

- **Рекурсивное сканирование** — находит все папки с `index.php` внутри добавленных корневых директорий (включая вложенные уровни).
- **Несколько корневых директорий** — например, основной диск и внешний `Arbitration`; каждая группа сворачивается отдельно.
- **Поиск по имени и пути** — фильтр в сайдбаре; опционально поиск по содержимому `index.php` (с кэшем).
- **Недавние и закладки** — быстрый доступ к часто используемым лендингам.
- **Вкладки** — несколько лендингов открыты одновременно; при смене вкладки автоматически переключается PHP document root.
- **Клавиатурная навигация** — `↑` / `↓` по списку, `Enter` для открытия.

### Превью PHP-лендингов

- Встроенный **PHP built-in server** на `127.0.0.1:8080`.
- Кастомный **`router.php`** — перед загрузкой `index.php` выставляет `$rawClick = true` и `$click = true`, чтобы лендинги с `die()` при отсутствии трекинг-параметров открывались в превью.
- **Кириллица в путях** — каждый сегмент URL кодируется через `encodeURIComponent`.
- Перезапуск сервера при переходе между лендингами из **разных** корневых директорий.

### Редактор кода

- **CodeMirror 5** с подсветкой PHP.
- Редактирование **любого `.php` файла** в папке лендинга (не только `index.php`) — выпадающий список файлов в шапке редактора.
- **Автосохранение** через ~1.2 с после последнего изменения.
- **История версий** — до 20 снимков на файл, хранятся на диске отдельно от `config.json`.
- **Поиск в коде** — `Cmd+F`, подсветка совпадений, `Enter` / `Shift+Enter` для навигации.
- **Перезагрузка с диска** — `Cmd+Shift+R` или кнопка ↻ (если файл меняли снаружи).

### Видео

- Кнопка **«Заменить видео»** — выбор исходника, перекодирование в **HLS** (`video/segments/playlist.m3u8`).
- Битрейт рассчитывается от длительности ролика (цель ~90 МБ).
- Диалог подтверждения с параметрами ffmpeg перед стартом.
- Прогресс-бар кодирования; предупреждение при попытке переключить лендинг во время encode.
- После успешной замены исходные `.mp4` / `.mov` в `video/` удаляются автоматически.

### Файлы и интеграция с macOS

- **Копировать папку** — в буфер обмена macOS (вставка через `Cmd+V` в Finder).
- **Показать в Finder** — открывает папку лендинга.
- **Дублировать лендинг** — копия с суффиксом `-copy`.
- **Автоочистка** — директории, недавние и закладки с путями, которых уже нет на диске (отключённый диск, удалённая папка), убираются автоматически.

### Терминал и MiMo Code

- Встроенный **xterm.js** + **node-pty** — shell в рабочей директории выбранного лендинга.
- Кнопка **MiMo** — проверка и установка [MiMo Code](https://mimo.xiaomi.com/) для AI-редактирования в терминале.

## Требования

| Компонент | Назначение |
|-----------|------------|
| **macOS** (Apple Silicon, arm64) | Целевая платформа сборки |
| **PHP** | `brew install php` — для встроенного сервера |
| **ffmpeg / ffprobe** | `brew install ffmpeg` — для замены видео |
| **Node.js 18+** | Только для разработки и сборки |

Приложение ищет бинарники в `/opt/homebrew/bin`, `/usr/local/bin` и через `command -v` в login shell.

## Установка

### Вариант A — готовый `.dmg` (рекомендуется)

1. Скачайте `PHP Browser-1.1.5-arm64.dmg` из [Releases](https://github.com/Nortsev/PHP-BROWSER/releases) *(или соберите сами — см. ниже)*.
2. Перетащите **PHP Browser** в **Программы**.
3. **Первый запуск:** правый клик → **Открыть** (приложение не подписано Apple).

### Вариант B — из исходников

```bash
git clone https://github.com/Nortsev/PHP-BROWSER.git
cd PHP-BROWSER
npm install
npm start
```

> **Важно:** для запуска используется `env -u ELECTRON_RUN_AS_NODE electron .` — это уже прописано в `npm start`. Без сброса `ELECTRON_RUN_AS_NODE` Electron может стартовать в неправильном режиме.

## Первый запуск

1. Откройте приложение.
2. В сайдбаре раскройте блок **«Директории»**.
3. Нажмите **«+ Добавить директорию»** и укажите папку с лендингами (например, `/Volumes/Arbitration/lander`).
4. Дождитесь сканирования — в списке появятся все папки с `index.php`.
5. Кликните по лендингу — превью откроется в правой части окна.

По умолчанию список директорий **пустой** — приложение не привязано к конкретному пути на диске и нормально работает на любом Mac.

## Руководство пользователя

### Добавление и удаление директорий

- **Добавить:** «+ Добавить директорию» → системный диалог выбора папки.
- **Удалить:** кнопка `×` рядом с путём в списке директорий.
- Если диск отключили или папку удалили вручную — запись исчезнет из списков при следующем сканировании (toast: «Убрано из списков (нет на диске)»).

### Работа со списком лендингов

- Группы сворачиваются по корневой директории (заголовок `lander (850)`).
- **Правый клик** на лендинге: Открыть · Копировать папку · Показать в Finder.
- **Закладка** — кнопка `☆` в панели адреса или звёздочка в списке.
- **Поиск «в коде index.php»** — чекбокс под строкой поиска; ищет по содержимому файлов (с задержкой и кэшем).

### Редактор

1. Выберите лендинг.
2. Нажмите **«Редактировать»**.
3. В выпадающем списке выберите файл (`index.php`, `config.php`, …).
4. Правки сохраняются автоматически; превью обновляется при сохранении `index.php`.
5. **История** — выпадающий список «История» для отката к предыдущей версии файла.

При переключении файла с несохранёнными изменениями появится запрос подтверждения.

### Замена видео

1. Откройте лендинг с HLS-структурой (`video/segments/playlist.m3u8`).
2. **«Заменить видео»** → выберите исходный ролик.
3. Подтвердите параметры кодирования в диалоге.
4. Дождитесь завершения прогресс-бара.

Перед encode приложение проверяет наличие ссылок на видео в `index.php` и `playlist.m3u8`.

### Терминал

- **«Терминал»** в панели адреса — открывает shell в папке лендинга.
- **MiMo** — установка/запуск MiMo Code в этой же директории.
- **Новая** — перезапуск сессии; **Очистить** — очистка экрана.

### Вкладки

- Кнопка **`+`** — новая вкладка с тем же лендингом (или пустая).
- Закрытие вкладки — `×` на вкладке (минимум одна вкладка всегда остаётся).
- Смена вкладки переключает активный лендинг и при необходимости PHP root.

| Клавиша | Действие |

## Конфигурация

Все данные хранятся в каталоге Electron userData:

```
~/.local/share/php-brauzer/
├── config.json              # директории, закладки, недавние
└── editor-history/
    └── <hash>/              # история правок по лендингу+файлу
        ├── meta.json
        └── 0001.php …
```

### Формат `config.json`

```json
{
  "directories": ["/path/to/lander"],
  "bookmarks": [
    { "dir": "/path/to/lander", "relativePath": "Архив/Erovitan", "name": "Erovitan" }
  ],
  "recents": [
    { "dir": "/path/to/lander", "relativePath": "Vizonic", "name": "Vizonic", "openedAt": 1719840000000 }
  ]
}
```

### Объект лендинга (Folder)

```js
{
  name: "Erovitan",           // имя папки
  dir: "/Volumes/Arbitration/lander",  // корневая директория сканирования
  relativePath: "Архив/Диман/Erovitan" // путь от dir; "" если index.php в корне dir
}
```

### URL превью

```
http://localhost:8080/<encoded-segments>/index.php
```

Каждый сегмент пути кодируется отдельно — кириллица и пробелы работают корректно.

## Архитектура

```
main.js          — Electron main: PHP-сервер, IPC, ffmpeg, сканирование, pty
preload.js       — contextBridge → window.api.*
index.html       — UI: сайдбар, webview, редактор, терминал
features.js      — вкладки, закладки, недавние, поиск по коду
editor.js        — обёртка CodeMirror (SourceEditor)
terminal-panel.js — xterm.js + IPC к node-pty
router.php       — роутер для php -S (обход die() в лендингах)
```

### PHP-сервер

- Один **document root** в момент времени — при смене корневой директории сервер перезапускается.
- Порт: **8080** (`127.0.0.1`).
- Health-check после старта; ошибки → toast «PHP-сервер недоступен».

### Безопасность редактора

- Относительные пути к PHP-файлам проверяются на `..` и выход за пределы папки лендинга.
- Редактируются только файлы с расширением `.php`.

### Что не попадает в git

```
node_modules/
dist/
*.dmg
```

## Решение проблем

### «Добавьте директорию с лендингами»

Нормально при первом запуске. Добавьте папку через «+ Добавить директорию».

### «PHP не найден»

```bash
brew install php
which php   # должен показать путь, например /opt/homebrew/bin/php
```

Перезапустите приложение.

### «Директория не найдена»

Путь в config есть, но папки на диске нет (отключён внешний диск). Подключите диск или удалите запись — она уберётся автоматически при сканировании.

### `ERR_CONNECTION_REFUSED` в превью

PHP-сервер ещё не поднялся или упал (занят порт 8080). Подождите секунду и нажмите ↻. Проверьте: `lsof -i :8080`.

### Превью пустое / `die()` в лендинге

Убедитесь, что используется встроенный `router.php` (входит в сборку). Он эмулирует клик для обхода защиты без GET-параметров.

### ffmpeg не найден

```bash
brew install ffmpeg
```

Кнопка «Заменить видео» требует оба бинарника: `ffmpeg` и `ffprobe`.

### Приложение не открывается после скачивания dmg

macOS блокирует неподписанные приложения. **ПКМ → Открыть** при первом запуске или:  
`xattr -cr "/Applications/PHP Browser.app"`

### Редактор показывает старый текст

Файл изменили снаружи (MiMo, другой редактор). Нажмите ↻ в шапке редактора или `Cmd+Shift+R`.
